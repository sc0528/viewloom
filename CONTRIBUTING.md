# Contributing to Viewloom

Viewloom is currently validating the creator problem before expanding the implementation. The most useful contribution is specific feedback about how creators compare content performance and which parts of that workflow should be automated.

- Use the [creator feedback form](https://github.com/sc0528/viewloom/issues/new?template=creator-feedback.yml) for product feedback.
- Open a regular issue for reproducible bugs or documentation problems.
- Do not post channel identifiers, private analytics, OAuth credentials, tokens, or screenshots containing private data.

Before proposing a code change, open an issue describing the user problem it solves. This keeps the early project focused on validated needs.
